# Contexto del Proyecto

Aquí debes agregar la información contextual necesaria para el análisis de historias de usuario.

## Archivos requeridos

Crea los siguientes archivos en esta carpeta:

### 1. `contexto-funcional.md`
Información de negocio, dominio y reglas operativas:
- Descripción del proyecto/producto
- Dominio y área de negocio
- Reglas de negocio aplicables
- Actores/roles principales
- Procesos de negocio relevantes
- Restricciones y limitaciones del negocio

### 2. `contexto-tecnico.md`
Stack tecnológico, arquitectura e integraciones:
- Stack de tecnologías (lenguajes, frameworks, bases de datos)
- Arquitectura del sistema (alto nivel)
- Integraciones externas y APIs
- Entornos (desarrollo, staging, producción)
- Restricciones técnicas y limitaciones
- Equipo (roles, capacidad por sprint)

## Cómo usar

1. Lee las plantillas en el proyecto
2. Documenta el contexto específico de tu proyecto
3. Los agentes usarán esta información para enriquecer el análisis de las historias de usuario
